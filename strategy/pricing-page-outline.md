# Vordr pricing page outline

## Goal

Make the pricing page explain the packaging clearly without making the open-source edition feel fake or making Enterprise sound like security was paywalled.

## Suggested top-of-page message

**One product. Three ways to buy it.**

- **Self-Hosted** gives you the core product.
- **Cloud** removes the operational burden.
- **Enterprise** adds organizational control.

## Section 1 — Packaging cards

### Card 1 — Self-Hosted
- Headline: Open-source self-hosted
- Positioning: Run the core product yourself
- CTA: Start self-hosted / View on GitHub
- Short bullets:
  - Core monitoring, alerts, dashboards, logs, transactions, and agents
  - BYOK AI
  - Community / docs support

### Card 2 — Cloud
- Headline: Managed by Vordr
- Positioning: Same product, less operational overhead
- CTA: Start trial / Book demo
- Short bullets:
  - Managed control plane
  - Included AI usage / credits
  - Easier onboarding, upgrades, and backups
  - Shared team usage

### Card 3 — Enterprise
- Headline: Organizational control
- Positioning: Identity, governance, support, and private deployment options
- CTA: Talk to sales
- Short bullets:
  - SSO / SAML / SCIM
  - Audit logs and advanced RBAC
  - Private deployment options
  - SLA / onboarding / support

## Section 2 — Feature comparison matrix

Use a matrix with a few rows buyers actually care about:

- Core monitoring
- AI model options
- Included AI credits
- Managed hosting
- Workspaces and RBAC
- SSO / SCIM / SAML
- Audit logs
- Private deployment
- Support level

Avoid an absurd 70-row table.

## Section 3 — Pricing philosophy

Explain what each edition is priced around.

### Self-Hosted
- Open-source core
- Monetization is not the goal here; adoption and trust are

### Cloud
Price around:
- monitored nodes / infrastructure footprint
- included AI usage
- maybe team size at higher tiers

Avoid pricing by alert count.

### Enterprise
Price around:
- deployment shape
- seat/team complexity
- support expectations
- private deployment / compliance scope

## Section 4 — FAQ

Answer these directly:

- Is the self-hosted edition actually useful on its own?
- Can I bring my own AI provider key?
- What is included in Cloud versus Enterprise?
- Can Enterprise be privately deployed?
- Do I need Enterprise just for basic security?

## Copy rules

- Do not describe the open-source edition as "community" in a dismissive way.
- Do not make Enterprise sound like basic secure deployment is paywalled.
- Do not oversell roadmap items as if they are fully mature if they are still foundations.
